## 标准概述

### IATF 16949:2016 简介

IATF 16949是汽车行业质量管理体系标准，由IATF(国际汽车工作组)发布，是ISO 9001在汽车行业的补充要求。

### 适用范围

- 顾客指定的汽车相关产品的设计/制造
- 零件、材料或热处理部件的生产
- 增值的过程（如：电镀、涂装）

### 核心工具

| 工具 | 全称 | 用途 |
|------|------|------|
| APQP | 产品质量策划 | 项目管理 |
| FMEA | 失效模式及影响分析 | 风险分析 |
| MSA | 测量系统分析 | 测量系统评估 |
| SPC | 统计过程控制 | 过程监控 |
| PPAP | 生产件批准程序 | 顾客提交 |
| Control Plan | 控制计划 | 过程控制 |

## 十大条款解析

### 4. 组织环境

**要求**:
- 确定内外部因素
- 识别相关方及其要求
- 确定质量管理体系的边界

**APQP关联**:
- 项目立项需考虑组织环境
- 识别相关方(顾客、监管机构)

### 5. 领导作用

**要求**:
- 质量方针与目标
- 职责权限
- 资源提供

**APQP关联**:
- 管理层支持是APQP成功关键
- 资源配置直接影响项目进度

### 6. 策划

**要求**:
- 风险与机遇识别
- 质量目标策划

**APQP关联**:
- FMEA是风险识别的核心工具
- 质量目标需分解到APQP阶段

### 7. 支持

**要求**:
- 资源能力培训
- 文件化信息
- 知识管理

**APQP关联**:
- APQP团队能力要求
- FMEA/Control Plan文件控制

### 8. 运行策划

**要求**:
- 顾客导向过程识别
- 产品设计要求
- 过程设计要求

**APQP关联**:
- 阶段1识别COP
- 阶段2/3对应设计和过程开发

### 8.5 生产和服务提供

**要求**:
- 控制计划执行
- 首件/末件确认
- 不合格品控制

**APQP关联**:
- 控制计划是核心输出
- PPAP后进入量产阶段

### 8.6 产品和服务的放行

**要求**:
- 检验/验证
- 顾客批准(如PPAP)

**APQP关联**:
- PPAP是强制顾客批准

### 8.7 不合格输出的控制

**要求**:
- 识别与隔离
- 文件化
- 顾客通知(如需要)

**APQP关联**:
- APQP过程中问题追踪
- 8D报告

### 9. 绩效评价

**要求**:
- 顾客满意度
- 内部审核
- 管理评审

**APQP关联**:
- 项目成功标准
- 阶段5反馈

### 10. 改进

**要求**:
- 不符合与纠正措施
- 持续改进

**APQP关联**:
- 经验教训总结
- 持续改进计划

## 与APQP关联的关键要求

### 8.3.1 产品设计开发策划

| 要求 | APQP对应 |
|------|----------|
| 设计策划 | 阶段1-2 |
| 多方论证方法 | 项目团队 |
| 同步工程 | 跨职能协作 |
| 风险分析(FMEA) | DFMEA |

### 8.3.3 产品设计输出

| 要求 | APQP对应 |
|------|----------|
| DFMEA | DFMEA文档 |
| 设计验证 | DVP&R |
| 特殊特性 | 特殊特性清单 |
| 设计评审 | DR1/DR2/DR3 |

### 8.3.4 产品设计验证

- 设计与样品验证
- 设计与过程确认

### 8.4.2 类型和程度控制

- 供应商开发要求
- 二方审核

### 8.5.1 生产和服务提供的控制

| 要求 | APQP对应 |
|------|----------|
| 控制计划 | 控制计划文件 |
| 作业指导书 | SOP |
| 预防性维护 | TPM计划 |
| 异物控制 | 清洁度控制 |

### 8.5.1.1 控制计划

**强制要素**:
- 产品/过程特殊特性
- 关键过程参数
- 检验/验证要求
- 不合格反应计划

### 8.5.6 过程变更管理

- 过程变更通知顾客
- 必要时重新PPAP

### 8.6 产品和服务的放行

- 100%检验(特殊特性)
- 顾客批准要求

### 8.7.2 顾客通知

以下情况需通知顾客:
- 不合格品发货
- 过程变更
- 供应商变更

## 审核要点

### APQP审核清单

| 审核点 | 检查项 |
|--------|--------|
| 项目立项 | 有项目章程、团队、RACI |
| 设计开发 | DFMEA完整、DR记录 |
| 过程开发 | PFMEA完整、控制计划批准 |
| 验证 | PPAP提交、顾客批准 |
| 量产移交 | 文件完整、培训完成 |

### 常见审核问题

1. **FMEA不完整**
   - 团队未参与
   - 措施未闭环

2. **控制计划不符**
   - 与PFMEA不一致
   - 特殊特性未标识

3. **PPAP文件缺失**
   - 未按要求等级提交
   - 样品与图纸不符

4. **过程能力不足**
   - Ppk未达标
   - SPC未实施

## 常见不符合项

### 按条款分类

| 条款 | 常见不符合 |
|------|-----------|
| 8.3.2 | DFMEA未覆盖所有功能 |
| 8.4.2 | 供应商FMEA缺失 |
| 8.5.1.1 | 控制计划与PFMEA不一致 |
| 8.5.1.2 | 首件检验未执行 |
| 8.6 | 追溯性缺失 |
| 9.1.1 | 顾客满意度数据缺失 |

### 严重度分级

| 级别 | 定义 | 示例 |
|------|------|------|
| 严重 | 系统性问题 | 无控制计划 |
| 重要 | 明显缺陷 | FMEA不完整 |
| 轻微 | 局部问题 | 文件格式 |
